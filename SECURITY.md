# Security Policy

## Supported Versions

The project is pre-1.0. Security fixes are best-effort on the latest `main` branch.

## Reporting a Vulnerability

Do not open public issues for security vulnerabilities.

Create a private security report through GitHub Security Advisories when available.
If private advisories are not enabled yet, open an issue titled
`security: private disclosure requested` and do not include exploit details.
Maintainers should then provide a private channel before receiving details.

Please include:

- A clear description of the vulnerability.
- Reproduction steps or proof-of-concept.
- Affected versions/commit hashes.
- Suggested mitigation, if known.

You can expect an acknowledgment within 72 hours and regular follow-up until resolution.
